# Security Policy

## Supported Versions

Security updates are provided for the latest minor release.

## Reporting a Vulnerability

If you discover a security issue:

1. Do not create a public GitHub issue.
2. Email security@twofold.tech with:
   - A detailed description
   - Steps to reproduce
   - Impact assessment
3. You will receive an acknowledgement within 48 hours.

## Handling Sensitive Data

- Never commit credentials or API tokens.
- Use a secrets manager (Vault, AWS Secrets Manager).
- Rotate tokens at least quarterly.

## Disclosure

We follow responsible disclosure and will credit reporters unless anonymity is requested.
